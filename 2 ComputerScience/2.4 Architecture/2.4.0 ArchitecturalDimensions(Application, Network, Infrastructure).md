# Architectural Dimensions: Application, Network, Infrastructure

1-tier, 2-tier, 3-tier, N-tier?

## Software (Codebased Application) Architecture

* Monolithic — единое приложение со всеми компонентами в одном процессе.

* Service-Oriented (SOA) — крупные сервисы с централизованной шиной (ESB).

* Microservices — множество независимых мелких сервисов (каждый за свою бизнес-возможность).

![Software (Codebased Application) Architecture](https://github.com/eldaroid/pictures/blob/master/iOSWiki/ComputerScience/EvolutionofSoftwareArchitecture.png?raw=true)

Эволюция: Monolithic (2000-е) -> SOA (середина 2000-х) -> Microservices (2010-е).


## Network interaction architecture

* Client-Server (Клиент-Сервер) - Классическая централизованная модель: клиент запрашивает, сервер отвечает. Основа веба и API.

> Нужна простота и контроль -> Client-Server.

* Peer-to-Peer (P2P, Одноранговая) - Полностью децентрализованная модель: все узлы равны и обмениваются данными напрямую. Используется в файлообмене и блокчейне. 

> Нужна устойчивость и децентрализация -> P2P. 

![ClientServerP2P](https://github.com/eldaroid/pictures/blob/master/iOSWiki/ComputerScience/ClientServerP2P.png?raw=true)

* Event-Driven/Event-Based (Событийно-ориентированная) - Реактивная модель: компоненты общаются через события (сообщения), а не прямые запросы. Для реального времени и сложных потоков данных.

> Нужна гибкая реакция на события и масштабируемость -> Event-Driven.

![EventBasedEventDriven](https://github.com/eldaroid/pictures/blob/master/iOSWiki/ComputerScience/EventBasedEventDriven.png?raw=true)

* API Gateway - единая точка входа для всех клиентских запросов к вашей системе.

## Infrastructure Architecture

* Web Application (Веб-приложение)

* Web Server (Веб-сервер)

* Application Server (Сервер приложений)
